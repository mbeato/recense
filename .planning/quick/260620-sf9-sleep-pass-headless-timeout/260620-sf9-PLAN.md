# Quick Task 260620-sf9: Sleep-pass headless timeout

## Problem

The scheduled sleep pass (`sleep-pass-cli.ts` → `runConsolidation` → `consumePendingCorpusMarkers` → `generateCorpusDocs`) runs headless `claude -p` calls at the **120s default** (`DEFAULT_TIMEOUT_MS` in `src/model/claude-headless-client.ts:85`). Phase 32-02 introduced **landing docs**, which are much larger than schema chapter docs — the `usage` landing doc has 148 citations and takes ~256s to generate. Under the 120s default the headless call is SIGKILLed → returns empty (`claude-headless-client.ts:326`) → `doc-generator.ts:357` declines to persist → **the landing doc never materializes on the scheduled deferred path** (32-03's primary trigger, D-03).

Every other corpus entry point already raises the timeout to 600s when unset:
- `src/adapter/generate-doc-cli.ts:129`
- `src/adapter/generate-corpus-cli.ts:85`
- `src/adapter/ingest-project-cli.ts:606,777`

The sleep pass is the one path that does not — so inline `ingest-project --consolidate` works but the scheduled deferred flow silently drops large docs.

Live evidence (2026-06-20, /tmp copy of live brain): same `usage` doc — empty under the sleep pass (120s), generated to 24,064 chars / 148 citations via the standalone CLI (600s). Clean A/B isolating the timeout.

## Tasks

### Task 1 — Raise the headless timeout default in the sleep-pass entry
- **files:** `src/adapter/sleep-pass-cli.ts`
- **action:** At the top of `main()` (before `runConsolidation`), add the same guard the other CLIs use: if `RECENSE_CLAUDE_HEADLESS_TIMEOUT_MS` is unset, default it to `600000`. Conditional so an explicit env override still wins.
- **verify:** `grep -n RECENSE_CLAUDE_HEADLESS_TIMEOUT_MS src/adapter/sleep-pass-cli.ts` shows the guard; `npm run build` clean.
- **done:** Sleep pass runs headless calls at 600s, matching the other corpus entry points; large landing docs generate on the deferred path.
