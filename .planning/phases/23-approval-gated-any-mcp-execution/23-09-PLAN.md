<objective>
Close GAP-01 (HIGH, D-43 / self-confirmation): HITL approval-gate audit episodes currently land as `source='http'`, `origin='observed'`, `consolidated=0` — indistinguishable from ordinary ingested memory and consolidation-eligible. Because audit content embeds the system's own retrieved tool RESULTS, the next sleep pass can extract and strengthen beliefs from them — the exact "never let inferred output strengthen a fact" hole the `source:'hitl'` intent (ACT-03 / H-12) was meant to close.

Fix: (1) thread a validated `source` through `POST /v1/add` → `ops.add` so the client's `hitlEpisode()` lands `source='hitl'`; (2) exclude `source='hitl'` from consolidation/extraction so audit records are a first-class, queryable, NON-consolidatable audit trail; (3) prove both against a REAL (non-mocked) serve add + a real consolidation pass.

Purpose: Restore the D-43 / ACT-03 must-have ("every decision lands as a source:'hitl' episode and no belief row is strengthened").
Output: validateSource helper + source-aware /v1/add + consolidator hitl exclusion + client source:'hitl' + a non-mocked provenance integration test.

Scope discipline (CLAUDE.md): surgical changes only. The fix MUST NOT weaken the D-05 'inferred'→'observed' clamp for any other source, and MUST NOT let a client spoof arbitrary source values — only the exact value 'hitl' is an accepted override; everything else falls back to the engine instance default.
</objective>
