<objective>
Build the `InsightReflector` deriver — the offline reflection step that synthesizes one
higher-order `type='insight'` node per qualifying stale schema cluster and wires it into
sleep-pass Phase C between `corpusPromoter.promote()` and `runEvictionSweep()`. The reflector
reuses the Phase-28 mass gate + noise filter for selection, the doc-staleness predicate to act
ONLY on stale/new clusters (no per-pass LLM re-run), the `doc-generator` judge-tier read-only
generate+verify path for synthesis, and the lifecycle-exempt single-writer pattern for the
node + `derived_from` edge + `node_insight` sidecar write. Self-confirmation holds by
construction (inferred origin + read-only-over-members) and is proven by a RED-under-injection
sentinel.

Purpose: turns "reasons over schemas" into a durable materialized engine mechanism (REFLECT-01)
without ever strengthening evidence from inferred output (D-43 / SC3).
Output: insight-reflector.ts, insight-generator.ts, Phase C wiring, the sentinel + no-op +
dissolution test suite.
</objective>
