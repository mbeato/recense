<objective>
Fold insight surfacing into recall: when schema resolution (Case A/B) lands on a schema that
has a live, non-stale, query-matching insight, return that single precomputed insight string
IN PLACE OF the K=20 `abstracts`-member neighborhood — one mode OR the other per query, never
both. Otherwise fall through to today's neighborhood assembly unchanged. The path is LLM-free
(reuses the already-computed query embedding + the resolved schema; freshness is a flag/timestamp
check, no synthesis at recall time), gated behind the `insightSurfacingEnabled` dark default so
the merge is a zero-behavior-change no-op until the 38-04 eval proves the compose-token win.
Falsified/tombstoned members never surface their dependent insight (the freshness gate stops
stale-insight self-confirmation).

Purpose: REFLECT-02 — recall returns a precomputed insight, shrinking the compose payload from
~K facts to one string, without ever synthesizing at recall time or strengthening anything.
Output: the recall insight-surfacing branch + its hit/miss/stale/no-mutation/flag-off test suite.
</objective>
