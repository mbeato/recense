# Phase 39.1 Plan 01: Hub/Subject Doc Generator Layer Summary

Hub + subject doc generators with D-37-gated gather, navigable linked index, and corpusSubjectDriftThreshold config knob.

## What Was Built

Three tasks delivered the complete generation layer for the D-01/D-04 doc taxonomy restructure:

**Task 1 — gatherFactsForSubject (doc-gather.ts, commit `641de4f`):**
Added `gatherFactsForSubject(deps, params, opts)` that gathers the union of `abstracts` members across ALL `schemaIds` in a subject's schema set. Mirrors `gatherFactsForSchema`'s three-source structure (spine + centroid-seeded semantic breadth + entity-hop) but spans multiple schemas. D-37 firewall applied on every member row (`n.type IN ('fact','entity') AND n.tombstoned = 0 AND n.origin != 'inferred'`) — prevents self-confirmation per D-43 / Pitfall 3. Spine tagged `'scope'` (identical to schema gather — no downstream changes needed). `GatherSubjectParams` type exported alongside.

**Task 2 — generateDocForHub + generateDocForSubject + prompt builders (doc-generator.ts, commit `21782de`):**
Added four functions to the generation layer:
- `buildHubDocPrompt(scope, factBlock, siblings, subjectDocs)` — subjectDocs is typed `{name: string; docId: string}[]` (NOT `subjectNames: string[]`); the prompt template embeds `recense://doc/<docId>` tokens so every index entry becomes a doc_link ref (D-04 navigability).
- `buildSubjectDocPrompt(scope, subjectName, factBlock, siblings)` — frames subjectName as thesis (NOT a schema UUID label per D-02).
- `generateDocForHub(deps, scope, subjectDocs, opts)` — gathers scope facts, builds hub prompt, empty-guards, runs `verifyCitations` (shared, not duplicated). Read-only.
- `generateDocForSubject(deps, {scope, subjectName, schemaIds, centroid?}, opts)` — gathers via `gatherFactsForSubject`, builds subject prompt, same empty-guard + `verifyCitations` core. Read-only.
- `generateDocForSchema` left untouched (backward compat per RESEARCH A3).
- `verifyCitations` defined exactly once — both new generators reuse it; `grep -c "function verifyCitations"` returns 1.

**Task 3 — corpusSubjectDriftThreshold + test file (config.ts + test, commit `21782de`):**
Added `corpusSubjectDriftThreshold: number` to `EngineConfig` with a JSDoc comment matching the existing knob style, and `corpusSubjectDriftThreshold: 3` to `DEFAULT_CONFIG` (D-06 default 3; analogous to `highMass:10` shape). Created `tests/doc-generator-hub-subject.test.ts` (16 tests) covering all four assertion groups from the plan's acceptance criteria.
