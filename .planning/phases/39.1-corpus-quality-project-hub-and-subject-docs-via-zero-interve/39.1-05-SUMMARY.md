## Mid-execution: consolidation-stall bug FOUND + FIXED (FIX-STALL-01, commit `67b3ade`)

Driving the sleep pass for Task 2 surfaced a latent bug: the bulk ingest pushed the backlog to ~308 episodes, and `consolidator.ts` prefetched extractions for ALL eligible episodes in one blocking `await` (~38 min) before the per-episode commit loop — exceeding `LOCK_STALE_MS` (30 min). The hourly launchd agent reclaimed the "stale" lock and replayed from scratch with zero `markConsolidated` commits: an infinite stall loop. Small backlogs always fit the window, which is why it only appeared after this phase's bulk ingest.

Fix: chunked the prefetch into batches of 20 (env `RECENSE_PREFETCH_EPISODE_BATCH_SIZE`) so progress is durable every ~2.5 min. tsc clean, 2147/2148 tests pass, confirmed LIVE (consolidated 2608→2609+). Full debug record: `.planning/debug/resolved/bulk-consolidation-stall.md`. This was a real latent engine bug that would have bitten any large ingest — the validation gate earned its keep.
