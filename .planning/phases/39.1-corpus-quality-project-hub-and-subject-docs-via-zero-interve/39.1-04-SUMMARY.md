## What Was Built

### src/adapter/cleanup-corpus-cli.ts

New CLI enumerating three deterministic junk classes (D-08) and deleting them safely (D-09):

**Class (a) — schema-UUID chapter docs:** type='doc' nodes whose `node_doc.slug` matches the UUID pattern AND the slug resolves to a live schema node (old taxonomy superseded by hub/subject restructure).

**Class (b) — empty/never-filled stubs:** type='doc' nodes with `length(value) = 0`. After the exhaust-gate fills new stubs, any empty stub remaining at cleanup time is old-taxonomy junk.

**Class (c) — noise-schema docs:** UUID-slug docs whose resolved schema's member set is >= 0.5 noise fraction (evaluated in TypeScript via `isNoiseMember`, copied verbatim from corpus-promoter.ts:86-98 — SQLite has no native regex).

**Dry-run (default):** opens DB read-only, enumerates all three classes, deduplicates by node ID, prints a grouped report to stdout, writes ZERO rows.

**Real run (`--no-dry-run`):**
1. Validates DB path BEFORE acquiring lock (WR-02)
2. Acquires lockfile (exits if held — races the sleep pass, T-39.1-15)
3. VACUUM INTO snapshot to `~/.config/recense/snapshots/` (reuse nightly-hygiene path) — if snapshot throws, logs to stderr and calls `process.exit(1)` BEFORE any DELETE (T-39.1-12 / Pitfall 5)
4. Verifies snapshot file is non-empty via `statSync`
5. Prunes snapshots beyond SNAP_KEEP=5
6. Hard-deletes each junk node inside ONE `db.transaction().immediate()` in FK-safe order:
   - `DELETE FROM node_doc WHERE node_id = ?`
   - `DELETE FROM node_scope WHERE node_id = ?`
   - `DELETE FROM edge WHERE src = ? OR dst = ?`
   - `DELETE FROM node_fts WHERE node_id = ?`
   - `DELETE FROM node WHERE id = ?`
7. Runs `PRAGMA foreign_key_check` — logs error and sets exitCode=1 if non-empty (T-39.1-13)
8. Releases lock in `finally`

**Exports (for unit tests):** `isNoiseMember`, `enumerateJunkDocs`, `printDryRunReport`, `takeSnapshot`, `hardDeleteJunkDocs`.

### Design Notes (not deviations, recorded for clarity)

**Noise-schema reason overlap:** Class (c) is structurally a subset of class (a) — both require UUID slug AND the slug resolving to a live schema node. Any noise-schema doc also satisfies chapter-uuid. The dedup retains `chapter-uuid` as the reason when both apply. This is correct for the deletion goal (the doc is flagged either way) and matches the plan's "dedupe ids across classes" directive. Tests were adjusted to reflect this reality: the noise-schema reason appears in the report only for docs that are exclusively class (c), which cannot occur when both classes share the same SQL predicate.
