<objective>
Build the `DocGraphDeriver` — the offline, LLM-free service that projects the existing schema/fact graph down into a multi-level doc->doc graph (`doc_reference` + `doc_containment` edges). This is the primary deliverable of Phase 39.2 and the sole owner of both edge kinds (D-11).

Purpose: Today the corpus is a 2-level hub->subject star (`doc_reference` near-empty, 4 edges). The rich structure already exists one layer down (2018 `abstracts` edges + shared facts/entities; subjects carry `subject-schema-ids` meta). This plan surfaces it as edges — no new knowledge, just re-expression of what the graph already encodes.

Output: `src/consolidation/doc-graph-deriver.ts` (NEW) with `DocGraphDeriver` + `NoopDocGraphDeriver`, plus unit tests. Wiring into the sleep pass is Plan 02; this plan is the pure derivation engine + its tests.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| facts/entities -> derived doc-edges | The deriver reads source nodes and writes derived edges. The only "untrusted" input is graph content that could be inferred/tombstoned and must not launder into the signal. |

The deriver is offline (Phase C sleep pass), LLM-free, read-only over facts, and writes only derived doc-edges to local SQLite. The threat surface is intentionally minimal — the real guards are correctness invariants, encoded honestly below.
