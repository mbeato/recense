## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator CLI -> local DB | A founder-invoked command writes derived doc-edges to the local SQLite DB under the shared write lock. Single-tenant, local-only; the only input is `--db <path>` / `--dry-run` flags. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39.2-12 | Denial of Service | shared write lock | mitigate | acquireLock() before any write; releaseLock() in finally on every path including error/exit (WR-02: validate DB path BEFORE acquiring the lock so an early exit cannot leak it). Mirrors promote-corpus-cli. |
| T-39.2-13 | Tampering (concurrent writers) | doc-edge tables | mitigate | If the lock is held (sleep pass running), the CLI exits(0) without writing — no concurrent wipe-and-rebuild of the same edge kinds. |
| T-39.2-14 | Repudiation | unintended live mutation | mitigate | --dry-run computes + reports counts without entering the deriver's write transaction (Plan 01 contract), so the operator can inspect before committing. |

No NEW package installs (net-zero deps); the CLI reuses existing better-sqlite3 + adapter primitives — no supply-chain checkpoint required.
</threat_model>
