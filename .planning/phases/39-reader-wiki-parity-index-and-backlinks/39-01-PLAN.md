<objective>
Surface incoming references ("Referenced by" / what-links-here) in the reader, the WIKI-02 backlink affordance. Two cases per D-05: the doc view lists other live docs that link here (incoming `doc_link`/`doc_reference`/`doc_containment` edges), and the atom/fact view lists which docs cite that fact (the reverse `cites` walk the Phase 27 staleness route already uses).

Purpose: recense already stores every reverse edge (`idx_edge_dst`); the only gap vs the LLM Wiki is that the reader never SHOWS them. This is pure presentation parity — no engine change (WIKI-03).
Output: a read-only `GET /doc/backlinks` route, a `fetchBacklinks()` render hook in the reader, an atom-side surface in the detail panel, and the muted "Referenced by" styling.
</objective>

  WHERE kind IN ('doc_link','doc_containment','doc_reference')
  AND src IN (SELECT id FROM node WHERE type='doc' AND tombstoned=0)

Doc-node label resolution (stmtDocNodes, server.ts:153) — COALESCE(NULLIF(sch.value,''), nd.slug) AS label, JOIN node_doc nd, LEFT JOIN node sch ON sch.id = nd.slug AND sch.type='schema'.

stmtGetDoc (server.ts:170): SELECT n.id, n.value, nd.generated_at ... WHERE n.type='doc' AND ns.scope = ? AND tombstoned=0 — resolves a slug to a live doc row {id,value,generated_at}.

stmtCitedFacts (server.ts:228): the reverse-cites walk from a doc id used by /doc/staleness (server.ts:587-643) — the precedent for the atom-side direction.

resolveDocSlugById(rawId) (server.ts:206): exact-or-unique-prefix resolution of a doc node id → slug, returns null on miss.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39-01 | Tampering / Injection | `/doc/backlinks` slug/id/fact params | mitigate | slug sanitized via `.toLowerCase().replace(/[^a-z0-9-]/g,'').slice(0,64)` (reuse /doc precedent); id via resolveDocSlugById (exact/unique-prefix, 404 on miss); fact param validated to id-charset; all binds are parameterized prepared statements (no string-built SQL) |
| T-39-02 | Information Disclosure | viz HTTP listener | accept | server binds loopback + Host-header loopback guard (returns 403) already in place; single-tenant local-only; no PII beyond what the reader already serves |
| T-39-03 | Tampering / write-path | /doc/backlinks route | mitigate | route is GET-only, opens no new Database (db is `{ readonly: true }`), runs only SELECT prepared statements, makes no LLM/provider/embed call — WIKI-03 read-only posture enforced (acceptance grep: exactly 1 `new Database` in server.ts) |
| T-39-04 | XSS (Tampering) | reader/detail DOM render of doc labels | mitigate | all DB-sourced strings (label, slug) set via `.textContent` only, never innerHTML or template literals (T-10-12/T-27-08); click handlers call preventDefault and route through existing ctx.openReader |
| T-39-SC | Tampering | npm/pip/cargo installs | mitigate | no new dependencies introduced this plan (net-zero deps rule) — no install tasks, so no package-legitimacy checkpoint required |
</threat_model>
