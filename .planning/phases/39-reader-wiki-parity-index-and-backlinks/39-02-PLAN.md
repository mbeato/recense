stmtDocNodes (server.ts:153) — the ENTIRE data layer for /index, compiled once:
  SELECT n.id, n.type, n.value, n.s, n.c, n.origin, n.tombstoned, nd.slug,
         COALESCE(NULLIF(sch.value,''), nd.slug) AS label
  FROM node n
  JOIN node_doc nd ON nd.node_id = n.id
  LEFT JOIN node sch ON sch.id = nd.slug AND sch.type='schema' AND sch.tombstoned=0
  WHERE n.type='doc' AND n.tombstoned=0
Each row has { id, slug, label }. Schema-anchored docs have slug = a UUID (sch.value supplies the human label); project-scope docs have slug = a human string (e.g. 'tonos') and sch.value IS NULL → label falls back to slug. Live data (2026-06-21): 23 doc rows — 1 project-scoped (tonos), 22 schema-anchored (UUID slugs).

Server route conventions (server.ts:474-545 /doc, :587-643 /doc/staleness): GET-only, db is `{ readonly: true }`, statements compiled once in the outer closure, JSON via res.writeHead(200,{'content-type':'application/json'}) + res.end(JSON.stringify(...)), catch → res.writeHead(500)+res.end('internal error'), always return.

## Trust Boundaries

| browser → viz HTTP server | `/index` GET request crosses from the (loopback) client into the server |
| DB → index DOM | doc labels/slugs from the DB are rendered into the #index-panel DOM |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39-05 | Tampering / Injection | `/index` route | mitigate | route takes NO params (returns the full doc set) — no untrusted input reaches SQL; binds only the compiled stmtDocNodes; no string-built SQL |
| T-39-06 | Information Disclosure | viz HTTP listener | accept | loopback bind + Host-header loopback guard (403) already in place; single-tenant local-only; the index exposes only doc labels the reader already serves |
| T-39-07 | Tampering / write-path | /index route | mitigate | GET-only, opens no new Database (`{ readonly: true }`), runs one SELECT, no LLM/embed/write — D-01/D-02 read-only projection (acceptance grep: exactly 1 `new Database` in server.ts) |
| T-39-08 | XSS (Tampering) | #index-panel render of labels | mitigate | all DB-sourced strings (label, slug) set via `.textContent`; slug used in navigation passed through encodeURIComponent; no innerHTML with user/DB data (T-10-12/T-27-08) |
| T-39-SC | Tampering | npm/pip/cargo installs | mitigate | no new dependencies introduced (net-zero deps; inline SVG icons, vendored libs only) — no install task, no package-legitimacy checkpoint required |
</threat_model>
