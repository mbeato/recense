<objective>
Build the synchronous, lossless, curated WRITE path: `recense remember "<fact>" [--scope <s>]`. The command stores the fact VERBATIM as a curated node, then runs a synchronous "mini sleep-pass" — embed → retrieve top-k neighbor beliefs → judge contradiction (reuse the same pure `update-decision.ts` routing + `SQLiteConsolidationSink`) → update-in-place on contradiction, else insert. This in-place reconsolidation is the differentiator vs. appending to a flat file.

Purpose: Close the customer-zero "replaces MEMORY.md" promise on the WRITE side. Deliberate facts currently leak to native Claude Code `.md` memory because every existing write path is lossy (turn-capture→sleep-pass, import-memory). `remember` is the synchronous, verbatim, in-place-updating path.
Output: `src/adapter/remember-cli.ts` (new), the `remember` dispatcher case in `recense.ts`, and `test/remember-cli.test.ts`.

| Boundary | Description |
|----------|-------------|
| user CLI argv → engine | The `"<fact>"` verbatim string and `--scope`/`--db` flags are user-supplied and cross into a DB write |
| concurrent writers → write lock | The hourly sleep pass and `remember` both contend for `recense-sleep.lock` |
| judge transport → graph mutation | The non-deterministic headless Sonnet verdict drives tombstone+mint |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-01 | Tampering | concurrent sleep pass holding the lock | mitigate | acquireLock fast-fails; on false print a clear "lock held — try again" and process.exit(0) — no partial write, no corruption (lock is per-write only) |
| T-33-02 | Tampering | judge returns out-of-set/hallucinated best_candidate_id | mitigate | resolve best_candidate_id via store.getNode; if null/tombstoned/self → fall through to the INSERT path (never edge to a non-existent node — mirrors the T-FK-01 null-coerce fix); foreign_key_check asserted empty in tests |
| T-33-03 | Denial of Service | judge call (~2-5s) blocks the user | accept | explicit ~2-5s write is the documented contract (NOT the LLM-free hot path); single judge call, subscription-billed, ~$0 marginal |
| T-33-04 | Repudiation | in-place update silently drops the user's correction | mitigate | D-04 force-reconcile guarantees an explicit contradiction always at least reconciles; D-01 preview prints `updated: "<prev>" → "<now>"` so the change is visible, never silent |
| T-33-05 | Information disclosure | verbatim fact may contain secrets written to the DB | accept | single-tenant local DB; `remember` is a deliberate user write of content the user chose to store (no redaction by design — verbatim is the contract) |
| T-33-SC | Tampering | npm/pip/cargo installs | mitigate | net-zero new runtime deps (engine invariant) — no install task in this plan; nothing to audit |
