<objective>
Build the HTTP backend for the in-app settings panel (D-03, second of the three settings.json
consumers): GET /settings (effective config + persisted preset/overrides), POST /settings
(write settings.json, key-whitelisted, then return updated config), and GET /usage (the token
readout — rolling-30d headline + all-time total, broken down by feature_tag so each line maps
1:1 to a toggle, D-09/D-10). Routes live inside the existing viz `http.createServer` handler so
they inherit the 127.0.0.1-only bind and the DNS-rebinding 403 guard already in place.

The viz server's DB handle is read-only; settings writes go to the filesystem (settings.json),
the /usage read uses the read-only DB handle to aggregate the ledger.

Purpose: the routes the 44-06 frontend panel calls.
Output: three routes + tests in src/viz/server.ts.
</objective>
