| Boundary | Description |

| server JSON → DOM | settings/usage values rendered into the panel could carry injected strings |
| panel → POST /settings | user input sent to the local write route |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-44-19 | Tampering (XSS) | rendering server/user values | mitigate | textContent-only for all server/user-sourced strings + numbers (mirror reader.js 452-463); no innerHTML for dynamic values |
| T-44-20 | Elevation | new Electron IPC/preload | mitigate | NONE added — panel is plain frontend modules + HTTP routes (D-02/D-102); verified by no preload changes in files_modified |
| T-44-21 | Tampering | client sending core-disable | accept | client only offers whitelisted toggles + has no core toggle; server (44-05) whitelists keys and the D-12 loader is the backstop — client cannot gut the core even if hand-crafted |
</threat_model>
