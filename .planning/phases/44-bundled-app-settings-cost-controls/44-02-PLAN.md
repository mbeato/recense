<objective>
Make settings.json actually take effect on the offline pass (D-06). Today
`run-sleep-pass.ts` and `ingest-project-cli.ts` read `RECENSE_CORPUS_GEN` /
`RECENSE_CORPUS_GEN_MAX` straight from `process.env`, so a settings.json override is
ignored; and schema induction runs unconditionally so the Lite preset cannot turn it off.

This plan routes both call sites through `loadMergedConfig` (env still wins per D-05) and
gates `inducer.induceSchemas()` on `config.schemaInductionEnabled` so presets control the
optional layer. This is wiring on the existing offline pass — NOT a re-architecture.

Purpose: presets/toggles from 44-01 become observable behavior in the sleep pass.
Output: corpus-gen + schema-induction now read the merged config; env precedence preserved.
</objective>
